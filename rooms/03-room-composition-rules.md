# ROOMS — Room Composition Rules

## Purpose

This document defines how each room should be laid out so the scenes remain readable, animatable, and emotionally clear.

## Base Rule

A room is not a random cute illustration.
A room is a functional stage set.

The job of the composition is to make state and function obvious.

## Standard Room Formula

Each room should contain:
- 1 primary work zone
- 1 secondary support zone
- 1 personality cluster
- 1 clear character anchor
- 1 lighting logic
- 1 status marker position
- 1 activity zone that can be animated clearly

## Primary Work Zone

This is where the node’s role is most obvious.
It should be the room’s main focal area.

### Examples
- M4: wide desk, screens, dials, control surfaces
- M1: bench, machine, pipes, prep/work surface
- Jeep: dashboard, steering wheel, mounted tools, mobile panel

## Secondary Support Zone

This supports the main work zone without competing with it.

### Examples
- shelving
- backup tools
- storage
- pinned notes
- wall-mounted indicators
- spare containers or work materials

## Personality Zone

This is the lived-in part of the room.
It prevents the scene from feeling like sterile UI art.

### Examples
- mug
- plant
- hanging charm
- radio
- lunch container
- toy or odd little decorative object
- sticker cluster

## Character Anchor

The character needs a stable, readable place in the composition.

Usually:
- near center
- slightly offset toward the work zone
- framed so the face and body remain readable

The character should not be buried in props.

## Layering Model

Each room should roughly divide into:

### Background layer
- wall
- windows if any
- structural surfaces
- low-noise pattern or paneling

### Midground layer
- main furniture
- workstation
- shelving
- large props

### Character layer
- agent figure
- seat or vehicle placement if relevant

### Foreground layer
- desk objects
- tools
- small interactive items
- readable animation targets

## Wall/Floor Separation

Every room should clearly distinguish between wall and floor or interior front plane.
This matters because it gives the eye immediate orientation.

### Good examples
- control room wall vs desk platform
- workshop wall vs floor mat/bench zone
- car interior upper dash vs lower cabin area

## Symmetry vs Asymmetry

Rooms should generally be asymmetrical but balanced.

Use asymmetry to create life.
Use balance to preserve readability.

Too symmetrical and the room feels dead.
Too chaotic and the room becomes muddy.

## Prop Clustering Rule

Props should be grouped by meaning.
Do not scatter objects evenly across the room.

### Good clusters
- shelf cluster
- desk cluster
- machine cluster
- wall-device cluster
- comfort cluster

### Bad clustering
- tiny objects distributed randomly everywhere
- decorative items with no compositional hierarchy

## Negative Space Rule

Every room needs breathing room.
This is especially important because the art style is already dense and rounded.

Negative space helps:
- the character read clearly
- animations remain visible
- the room state be understandable

## Activity Zone Placement

Each room needs a visually obvious area where “work is happening.”
This is the place GSAP should be able to energize.

### M4 activity zone
- desk console
- central screen array
- control cluster

### M1 activity zone
- machine/workbench
- prep/work surface
- hot industrial focal point

### Jeep activity zone
- steering/dashboard cluster
- console panel
- mobile field interface

## Status Dot Placement

The status indicator should be easy to find but not dominate the illustration.
Best options:
- upper corner
- dedicated panel element
- wall-mounted light
- dashboard light

It should feel part of the room, not pasted on top as an afterthought.

## Down State Composition Consideration

The room has to support an offline/down state without redesign.
That means the base composition should leave room for:
- darkened lights
- cobweb placement
- locked door cue
- dead monitor cue
- stale atmosphere cues

## Room Readability Test

Each room should pass this test:

At a glance, can a viewer tell:
- what this room does
- who belongs in it
- where the work happens
- whether it is calm, active, or broken

If not, the composition is not working yet.

## Summary

A good ROOMS composition is:
- readable
- layered
- clustered
- lived in
- state-ready
- animation-ready
